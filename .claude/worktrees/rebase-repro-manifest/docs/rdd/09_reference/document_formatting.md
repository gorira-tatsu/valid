# Document Formatting Rules

- ドキュメントID: `RDD-0001-17`
- 目的: `docs/rdd` 全体の見出し、参照、関連IDの書式を統一し、読みやすさと差分レビュー容易性を確保する。

## 必須メタ情報

本文系ドキュメントは先頭付近に次を持つ。

- タイトル
- ドキュメントID
- バージョン
- 目的
- `関連ID`

README 系は簡略化してよいが、最低でも `関連ID` か `文書ID` のどちらかを置く。

## 参照ラベル

- `関連ID:` を正準ラベルとする。
- `参照元:` はその文書を読む入口だけを示す。
- `次に読む:` は必要なときだけ使う。
- `関連:` や `ID参照:` は新規追加しない。既存文書も順次 `関連ID:` へ寄せる。

## ID 記法

- Requirement: `REQ-001`
- Functional Requirement: `FR-001`
- Non Functional Requirement: `NFR-001`
- Epic: `A-1`, `C-4`
- PR: `PR-03`
- Document: `RDD-0001-12`
- KPI: `KPI-03`

ID を本文中で初めて使う箇所では、可能なら対応ドキュメントへのリンクを置く。

## リンク方針

- 章をまたぐ参照はリポジトリ相対または文書相対の markdown link を使う。
- 単なる列挙ではなく、読者が次に飛ぶ必要があるリンクだけを置く。
- `old/` 配下は現行仕様の根拠に使わない。

## セクション設計

長い本文は次を優先する。

1. 何を決める章か
2. 境界と責務
3. データ
4. 振る舞い
5. 失敗条件
6. 参照

この順にすると、設計判断が先に見える。
